# 身份证生成器 — 设计规格

**日期**: 2026-07-14
**技术栈**: HTML + CSS + JS（单文件）

---

## 1. 项目概述

一个数据模拟工具，在浏览器中生成符合中国身份证编码规范（GB 11643）的虚拟身份证信息。用户可选择性别、出生日期、省/市/区三级地址，系统自动生成合法的 18 位身份证号、对应性别的随机姓名，并以信息卡片形式展示。

## 2. 整体结构

单文件 `idcard-generator.html`，浏览器直接打开即用，零依赖。

| 区块 | 说明 |
|---|---|
| 顶部 - 控制面板 | 性别选择、出生日期选择器（年/月/日联动）、省/市/区三级联动下拉框、「生成」按钮 |
| 中部 - 结果卡片 | 结构化信息列表（姓名、性别、民族、出生日期、住址、身份证号），含一键复制按钮 |
| 底部 - 数据区 | 地区码表、常用姓氏和男/女性名字库，均为 JS 常量 |

视觉风格：简洁白底卡片式，居中单列布局。

## 3. 交互流程

```
用户调整选项 → 点击「生成」→ 校验 + 组装数据 → 渲染结果卡片
                        ↑
             性别 → 姓名筛选 + 顺序码第17位奇偶
             出生日期 → 日期码 7-14 位
             地区选择 → 地址码 1-6 位
             random() → 顺序码 15-17 位
             GB 11643 → 校验码 18 位
```

### 3.1 联动规则

- **性別联动**：随机姓名从对应性别名字库抽取；顺序码第 17 位奇数=男、偶数=女
- **出生日期联动**：月份改变时日的可选范围随之变化；闰年判断影响 2 月天数
- **地区联动**：选省→加载对应市列表，选市→加载对应区县列表，区县选定则地址码确定
- **校验码**：前 17 位按 ISO 7064:1983 MOD 11-2 自动计算

### 3.2 一键复制

身份证号旁设复制按钮，点击后将 18 位号码写入剪贴板，按钮文字短暂变为"已复制"（1.5 秒后恢复）。

## 4. 身份证号结构（18 位）

| 位置 | 含义 | 来源 |
|---|---|---|
| 1-6 | 地址码 | 用户选择的省/市/区对应编码 |
| 7-14 | 出生日期码 | 用户输入，格式 YYYYMMDD |
| 15-17 | 顺序码 | 随机生成 3 位数字；第 17 位奇偶与性别一致 |
| 18 | 校验码 | 前 17 位加权求和 mod 11，映射到 `1 0 X 9 8 7 6 5 4 3 2` |

**校验码算法**：
- 权重数组：`[7, 9, 10, 5, 8, 4, 2, 1, 6, 3, 7, 9, 10, 5, 8, 4, 2]`
- 前 17 位逐位 × 对应权重，求和
- 和对 11 取模，按映射表得到校验码字符

## 5. 数据结构

### 5.1 地区码表

三层嵌套对象，key 为中文名称，value 为子级对象或地址码字符串。结构示例：

```js
const regionData = {
  "北京市": {
    code: "110000",
    cities: {
      "市辖区": {
        code: "110100",
        districts: { "东城区": "110101", "西城区": "110102", ... }
      }
    }
  },
  ...
}
```

### 5.2 姓名库

- 常见姓氏：约 100 个（王李张刘陈杨黄赵周吴……）
- 男性常用名字字：约 150 个单字
- 女性常用名字字：约 150 个单字
- 生成规则：随机取 1 个姓 + 1~2 个随机常用名字字

### 5.3 民族

默认填写"汉族"，预置常见民族列表供扩展（汉、壮、满、回、苗、维吾尔、土家、彝、蒙古、藏等）。

## 6. 边缘情况处理

| 场景 | 处理方式 |
|---|---|
| 未选完地址即点生成 | 按钮置灰，下方提示"请先选择完整地址" |
| 闰年 2 月 29 日 | 日选择器动态限制：闰年可选 29 日，平年最多 28 |
| 不支持 Clipboard API | 选中号码文本，提示用户手动 Ctrl+C |
| 首次加载无默认结果 | 控制面板预填充默认值（男、1990-01-01、北京市-市辖区-东城区），并自动生成一次展示效果 |

## 8. 验收标准

1. 生成的身份证号 18 位，校验码通过 GB 11643 算法验证
2. 身份证号第 17 位奇数对应男性、偶数对应女性
3. 姓名性别与所选性别一致
4. 日期选择器正确处理闰年/平年及不同月份的天数
5. 地区三级联动正常，地址码与所选区县匹配
6. 复制按钮可将身份证号写入剪贴板
7. 不完整地址时生成按钮不可用
